# 코드 리뷰 체크리스트

## 1. 코드 품질

### 네이밍
- [ ] 변수명이 명확하고 의미를 전달하는가?
- [ ] 함수명이 동작을 정확히 설명하는가?
- [ ] 약어 사용을 최소화했는가?
- [ ] 일관된 네이밍 컨벤션을 따르는가? (snake_case/camelCase)

### 구조
- [ ] 함수가 단일 책임을 가지는가?
- [ ] 중복 코드가 없는가?
- [ ] 적절한 추상화 레벨을 유지하는가?
- [ ] 파일/모듈이 논리적으로 구조화되어 있는가?

### 가독성
- [ ] 코드가 자기 설명적인가?
- [ ] 복잡한 로직에 주석이 있는가?
- [ ] 매직 넘버를 상수로 정의했는가?
- [ ] 들여쓰기와 포맷팅이 일관적인가?

## 2. 에러 처리

- [ ] 예외 처리가 적절한가?
- [ ] 사용자에게 의미 있는 에러 메시지를 제공하는가?
- [ ] 에러 로깅이 충분한가?
- [ ] Try-except 블록이 너무 넓지 않은가?

## 3. 성능

- [ ] 불필요한 연산이 없는가?
- [ ] 루프 내 중복 계산을 피했는가?
- [ ] 데이터베이스 N+1 쿼리 문제가 없는가?
- [ ] 적절한 인덱싱을 사용하는가?
- [ ] 메모리 누수 가능성은 없는가?

## 4. 보안

- [ ] 사용자 입력을 검증하는가?
- [ ] SQL 인젝션 취약점이 없는가?
- [ ] XSS 공격에 안전한가?
- [ ] 민감한 정보가 하드코딩되어 있지 않은가?
- [ ] 인증/인가가 제대로 구현되어 있는가?

## 5. 테스트

- [ ] 주요 로직에 대한 테스트가 있는가?
- [ ] 엣지 케이스를 테스트하는가?
- [ ] 테스트가 실패할 수 있는가? (의미 있는 테스트)
- [ ] 테스트 커버리지가 충분한가?

## 6. 문서화

- [ ] 공개 API에 docstring/JSDoc이 있는가?
- [ ] 복잡한 알고리즘에 설명이 있는가?
- [ ] README가 최신 상태인가?
- [ ] API 문서가 업데이트되었는가?

## 7. 의존성

- [ ] 불필요한 의존성이 없는가?
- [ ] 의존성 버전이 명시되어 있는가?
- [ ] 보안 취약점이 알려진 패키지를 사용하지 않는가?

## 8. 형상 관리

- [ ] 커밋 메시지가 명확한가?
- [ ] 하나의 커밋이 하나의 논리적 변경을 담고 있는가?
- [ ] .gitignore에 불필요한 파일이 제외되었는가?
- [ ] 민감한 정보가 커밋되지 않았는가?

## 우선순위 가이드

### 🔴 Critical (즉시 수정)
- 보안 취약점
- 데이터 손실 가능성
- 서비스 다운 위험
- 치명적 버그

### 🟡 Warning (수정 권장)
- 성능 이슈
- 베스트 프랙티스 위반
- 잠재적 버그
- 유지보수성 문제

### 🔵 Info (개선 제안)
- 코드 가독성 개선
- 리팩토링 제안
- 문서화 보완
- 스타일 가이드 준수
